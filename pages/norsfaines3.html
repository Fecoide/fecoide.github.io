3.2 语法

3.2.1 帕迷和韦尔斯

经过前两节课, 我们能够注意到, 诺法语有状态组与关系组两大范畴. 而组内亦有分别: 简而言之, 我以语义完整度区分, 自足而完整者, 我称其为帕迷 POBNE ; 不完整而待补足者, 我称其为韦尔斯 VERS . 韦尔斯是抽象的, 先验的, 它们是精灵从甘第符雷捕捉到的; 而帕迷是经验的, 逻辑的, 它们是精灵实际可以绘制出的意识地图. 因为韦尔斯是原初之诗的碎片, 所以它天生渴望成为帕迷的一部分.

帕迷 韦尔斯
状态组 状态句(pr.) 状态词(vr.)
关系组 关系句(pa.) 关系词(va.)

3.2.2 化合

无论是状态组还是关系组, 都可作为一个独立的描述, 但单独的状态词或关系词是不完善或是不丰满的. 而状态句和关系句更完整, 但同样也可以配合使表达更*丰富*. 因此, 状态组需要同关系组配合使用, 以构成一个更大的组, 即化合 JIETHAIR . 化合的规则集如下:

设 Φ 为生成规则集, R 为状态组, A 为关系组. 包含至少以下一条规则:
  Φ₁: R × A → A;   Φ₂: A × R → R

这是极其不对称的规则, 而肖挪伊(SIANOËTTOS) [译注] 递归系统 的本质即不对称. 的在正式的对话中, 韦尔斯必须化合为帕迷, 这就是禁足的韦尔斯 LESAITTEVEACONAIR . 词的化合可以嵌套. 例如:

mzen-: 状态词 = "猫"
-ainfse: 关系词 = [+有灵], [+多次], [+参照]
-cobn-: 关系词 = "吃"
-ainx-: 关系词 = [+施事]
-er: 状态词 = 结构补位, 词典形, 无意义
mzen- + -ainfse → mzenainfse: 关系句 = "事件以多次靠近作为个体的猫的状态发生"
mzen- → mzêi-(历史语音变体), mzêi- + -cobn- → mzêicobn-: 关系词 = "猫吃东西", 状态未说明
mzêicobn- + -er → mzêicobner: 关系句 = "猫吃东西", 状态定义为 "未定义"
cobn- + mzen- → cobnemzen-: 状态词 = "已知事件 '吃' 与角色 '猫'", 关系未说明
cobnemzen- + -ainx- → cobnemzènainx-: 关系词 = "已知事件 '吃' 与角色 '猫', 角色为施事", 状态未说明
cobnemzènainx- + -er → cobnemzènainxer: 状态句 = "已知事件 '吃' 与角色 '猫', 角色为施事", 状态定义为 "未定义"

3.2.3 古典化合入门

化合规则非常复杂, 现已学习到的内容远远不足以理解 3.2.2 中的例子, 本章仅介绍一类最简单的古典化合. 古典化合即在生成状态-关系树(A-R 树)时, 依据时间顺序由左至右依次遍历并结合. 例如 mzenainfse 可以得到 A-R 树:

flowchart TD A(PA) --> B(VR) --> D[mzen-] A(PA) --> C(VA) --> E[-ainfse]

如上, 这是一个最简关系句的结构, 即由状态词干 + 关系词尾这两个韦尔斯构成, 其中关系词尾包含了视角场, 次数以及灵性三重范畴. 类似地, 一个最简状态句的结构为关系词干 + 状态词尾, 其中状态词尾包含了流向以及生命周期两大范畴. 所谓的 "词尾", 即语法化了的, 失去独立意义的位于帕迷尾部的韦尔斯, 我们称其为踵接词 BENHS ; 所谓的 "词干", 即尚未语法化或半语法化的韦尔斯, 它们至少仍然残留了一些独立的抽象语义, 我们称其为主干词 CORF .

主干词 踵接词
最简状态句 关系词 状态词 {流向, 生命周期}
最简关系句 状态词 关系词 {视角场, 次数, 灵性}

3.2.4 灵性的幽灵

我们知道, 关系句一定存在灵性范畴, 而状态句不一定. 仔细观察 3.2.3 的表格的踵接词列, 可以发现 "灵性" 这个范畴似乎是多余的, 它被归纳入关系句的踵接词范畴, 因此关系句的修辞负担看起来更重了, 这破坏了我们的*美感*与*直觉*. 事实上, 这个表格并不严谨, 灵性实际上是比韦尔斯化合句更上层的它的肖挪伊泰(SIANOËTTARS) [译注] "肖挪伊泰" 类似于集合中的父集合 的语法范畴, 而它会在韦尔斯上留下投影, 只是在关系句形态上体现得更明显罢了. 令人烦恼的是, 当我们试图通过观测定位灵性位于哪个韦尔斯时, 它往往不会独属于任何韦尔斯; 当我们试图在整个语句中追踪灵性如何实现时, 它总是把自己隐藏在具体的韦尔斯中. 当然, 灵性并非真的是幽灵, 只是因为肖挪伊内部无法完整描述肖挪伊本身的某个属性而已.

为了方便起见, 我们约定: 灵性的形态变化位于.关系踵接词上; 灵性的开关位于状态主干词上. 具体约定示例: 有状态主干词 mzêi- "猫", 其存在固有属性 "有灵", 故其滤镜视角下的单次形态为 mzens, 即在踵接词上选择了有灵的形态. 但约定并非本质, 我们不能把语言的月亮当成真实的月亮.

3.3 练习

判断正误:

1. 关系句是由状态主干词 + 关系踵接词这两个韦尔斯构成的. FALSE. 状态主干词 + 关系踵接词只是古典化合构成最简关系句的方式, 并非是构成关系句的唯一方式.

2. 一个帕迷是由多个韦尔斯构成的. FALSE. 本章只介绍了韦尔斯渴望组合构成帕迷, 并没有说一个元素不能既是韦尔斯又是帕迷, 也没有说一个帕迷不能自嵌套.

3. 古典化合不只适用于构成最简关系句; 实际上, 在帕迷的范围内, 古典化合构成的一定是关系句或状态句, 无论复杂与否. FALSE. 古典化合构成的可能是韦尔斯.

4. 古典化合不只适用于构成最简关系句; 实际上, 古典化合构成的要么是帕迷, 要么是韦尔斯, 无论复杂与否. FALSE. 古典化合构成的可能既不是帕迷也不是韦尔斯. 比如, 多个帕迷通过古典化合构成的可能是喜爱组, 伊古斯组等等.

5. 一个最简状态句的构成方式从逻辑上看与古典化合相同, 换言之, 最简状态句只能通过古典化合构成.FALSE. 最简的状态性状态句可以通过古典合成构成, 但是最简的关系性状态句存在厄尔托库斯效应, 不一定通过古典合成构成.

3.4 闪烁的特耳荷斯

诺法语有一个比较高频的韦尔斯 yey-. 古代法罗士人把这个韦尔斯翻译做 "伴侣, 朋友", 不知道他们怎么想的, 他们翻译的时候肯定不知道法罗士这个三千年宿敌也完全属于诺法语的 yey- 的概念. 它的单次滤镜形态是 yoi, 多次滤镜是 yeys. 但还有很奇怪的点,它的单次滤镜从形态上看显然是无灵的. 而理论上, 它来自上古诺法语的 *nik-, 这种整体发音舌位非常靠前的应该是有灵的概率更大, 甚至从纯粹的语义上也是偏向主动的, 也就是应该有 *nik-s > *yege, 但实际并非如此, 令人困惑.

现在是古代, 有一位生活在遥远的伊塔穆城邦内的一个诺法青年, 而伊塔穆城邦北面与法罗士隔着关隘相望. 有一次, 他的法罗士朋友问他: "先生, 您足够爱我吗?" 诺法青年回答: "当然. 在我的家乡话里, 你足以成为我的 yoi." 法罗士朋友继续问: "假若你我的城邦曾发生过数百年的战争, 仇敌们的刀剑在鲜血中相抵无数次, 直到今天, 你远远地望见在敌军方阵里的我 - 那时, 我还是 yoi 吗?" 诺法青年回答: "不仅是, 而且还是 yeys."

所谓由来, 历史, 未来, 便是岁月的一生
